# Developer Onboarding

## Purpose
Цей документ призначений для нового учасника команди, який починає роботу з документацією проєкту онлайн-платформи для навчання.

## Documentation Structure
- `README.md` — головна точка входу в документацію
- `architecture/SSD.md` — вимоги до системи
- `architecture/SDD.md` — архітектура системи
- `architecture/ISD.md` — інфраструктура та розгортання
- `quality/test-strategy.md` — підхід до тестування
- `quality/traceability-matrix.md` — зв’язок між вимогами і тестами

## Where to Start
1. Прочитати `README.md`
2. Ознайомитися з `SSD.md` для розуміння вимог
3. Прочитати `SDD.md` для розуміння архітектури
4. Переглянути `ISD.md` для розуміння середовища розгортання
5. Ознайомитися з документами якості

## Documentation Rules
- Не змінювати вимоги поза SSD
- Не описувати архітектуру поза SDD
- Не змінювати тестову простежуваність поза Traceability Matrix
- Кожна зміна документації повинна бути зафіксована у Git

## Versioning
Документація підтримується разом із розвитком системи і має власну версійність.
