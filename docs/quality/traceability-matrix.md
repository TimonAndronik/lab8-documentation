# Traceability Matrix

## Призначення документа
Цей документ містить матрицю простежуваності між вимогами SSD та тестовими сценаріями.

## Матриця простежуваності

Матриця простежуваності показує зв’язок між вимогами SSD і тестовими сценаріями. Вона потрібна для підтвердження того, що всі основні вимоги мають тестове покриття.

| Вимога | Опис вимоги | Тест |
|---|---|---|
| FR-01 | Реєстрація користувача | TC-01 |
| FR-02 | Автентифікація користувача | TC-02, TC-03 |
| FR-03 | Розмежування ролей | TC-05, TC-10 |
| FR-04 | Створення курсу | TC-04 |
| FR-05 | Редагування курсу | TC-04 |
| FR-06 | Перегляд каталогу курсів | TC-06 |
| FR-07 | Запис на курс | TC-07 |
| FR-08 | Перегляд навчальних матеріалів | TC-08 |
| FR-09 | Проходження тестування | TC-09 |
| FR-10 | Автоматичне оцінювання тесту | TC-11 |
| FR-11 | Перегляд результатів | TC-12, TC-13 |
| FR-12 | Адміністрування користувачів і курсів | TC-10 |

## Пояснення до матриці простежуваності

Матриця простежуваності є важливим елементом документації якості, оскільки вона демонструє логічний зв’язок між вимогами та перевірками. Якщо якась функціональна вимога не має жодного тесту, це означає ризик того, що реалізація цієї функції залишиться неперевіреною. Якщо ж один тест покриває кілька вимог, це дозволяє оптимізувати процес тестування, але вимагає чіткого опису очікуваних результатів.
